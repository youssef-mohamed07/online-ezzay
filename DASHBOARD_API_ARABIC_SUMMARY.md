# 📊 ملخص Dashboard API - نسخة عربية مختصرة

## 🎯 الهدف
عرض إحصائيات وبيانات الشحنات في لوحة التحكم (Dashboard)

---

## 🔗 الـ Endpoint

```
GET /wp-json/ezzy/v1/dashboard
```

**المصادقة:** Basic Auth (Consumer Key & Secret)

---

## 📦 البيانات المطلوبة

### 1️⃣ المؤشرات الأساسية (4 أرقام)

| الحقل | المعنى | مثال |
|------|--------|------|
| `total_shipments` | إجمالي الشحنات | `221` |
| `active_shipments` | الشحنات النشطة (لم تُسلّم بعد) | `127` |
| `delivered_shipments` | الشحنات المسلّمة | `94` |
| `warehouse_parcels_count` | الطرود في الصندوق/المستودع | `15` |

### 2️⃣ آخر الشحنات (اختياري)

```json
{
  "recent_shipments": [
    {
      "tracking_number": "y002",
      "status": "في الصندوق",
      "date": "2026-04-13"
    }
  ]
}
```

### 3️⃣ مؤشرات إضافية (اختياري)

أي رقم إضافي سيظهر تلقائياً:

```json
{
  "pending_shipments": 10,
  "cancelled_shipments": 3,
  "total_revenue": 150000
}
```

## 🔄 ماذا يحدث إذا لم ترسل البيانات؟

التطبيق سيحسبها تلقائياً من قائمة الشحنات:
- يجلب كل الشحنات من `/ezzy/v1/shipments`
- يحسب الأرقام محلياً

**لكن:** إرسال البيانات من الباك إند **أسرع وأدق** ✅

---

## 📝 ملاحظات

### ✅ يمكن استخدام أسماء بديلة:

```json
// كل هذه صحيحة:
{ "total_shipments": 221 }
{ "shipments_count": 221 }
{ "shipments": { "total": 221 } }
```

### ✅ يمكن إضافة data wrapper:

```json
{
  "data": {
    "total_shipments": 221,
    ...
  }
}
```

### ✅ الأرقام يمكن أن تكون:
- `int`: `221`
- `double`: `221.0`
- `string`: `"221"`
